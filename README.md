## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

このアイデアの原点は、自分自身が英語学習やプログラミング学習を１人でコツコツと続けてきた経験にあります。

私は、留学やワーキングホリデーをきっかけに、学習計画やTodoアプリを利用してタスク管理を行い英語学習に取り組んできました。

一人暮らしの社会人時代は、仕事や家事と並行しながら終業後や週末の隙間時間を利用して英語学習を行い、ワーホリ時代は慣れない海外で、不安や孤独になりながらも継続的にプログラミング学習を行なってきました。

しかし、英語やプログラミングのような学習は、必死に学習計画を立てて隙間時間で勉強したからといって、すぐに大きく伸びるものではなく成果もすぐに見えず、前に進んでいる実感が持てず、継続のモチベーションの管理が難しい経験をしました。

その中で、以下のように目標を達成できたら自分へご褒美を設定することが学習継続のモチベーションの１つとして有効でした。
```
例)
今週3回DMM英会話ができたら、友達とパブでビールを飲む
普段ならサボる時間帯に学習できたら、好きなクッキーを買う
```

---

### 2-2. なぜそれが「気になった」のか
- 自分の性格
	ゲームのように自分の成長が数字で可視化されることが楽しい
	完璧主義で、タスクを設定してできなかった時に罪悪感や自責の念を感じる

- 過去の経験
	小学校から高校までは、自主的に学習に取り組むという習慣が全くなかった。
	学習を始めた頃はサボったり、後回しにしたりしていたが、徐々に継続することで「できなかったこと」が、「できるようになる」瞬間に達成感を感じるようになった。
	「海外で自分自身で生活基盤を築く」「ヨーロッパを１人で旅行する」という、数年前の自分からは想像もできない体験をした。
- 大切にしている価値観
	１人で海外でも問題なく生活できるようになれたのは、コツコツと継続して工夫をしながら学習を続けた経験があるからだと信じており「工夫」をして継続することに価値があると感じている。

上記のような経験から、自分にとって学習とは、単に知識を増やすためのものではなく、自分の行動や工夫によって人生の選択肢を広げられるものだと感じています。

だからこそ、学習を継続するための「工夫」や、努力を前向きに受け取れる仕組みに価値があると考えています。

このように、「やろうと決めたことに取り組めた」「いつもならサボる時間に着手できた」という行動を評価し、ゲームのように前向きに続けられる仕組みをアプリとして作りたいと考えたのが、このアイデアの出発点です。

そのため次に、自分の体験やヒアリング内容をもとに、表面的に見えている困りごとと、本当に解決したい課題を整理します。

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

ヒアリングから、働きながら学習している社会人には以下のような困りごとがあると分かった。
```
- 学生のように、毎日同じ時間にまとまった学習時間を確保することが難しい
- 仕事の疲れから、「今日は勉強しよう」と思っていても後回しになりやすい
- 一人で学習しているため、周囲からの強制力が弱く、簡単にサボれる
- TOEIC専用アプリ、プログラミング教材、問題集、動画教材など、学習媒体が分散している
- Notion、Todoアプリ、Excel、カレンダーアプリは多機能だが、管理すること自体が負担になりやすい
- 目標を設定して学習に取り組んでも、期待したほど成果が見えず、落ち込むことがある
```

ヒアリング内容からより大きな問題は、忙しい中で学習に取り組んでいても、その努力や前進が実感しづらいことにあると考えた。

---
### 3-2. 本当に解決したい課題は何か

```
TOEIC、プログラミング、資格学習などは、成果が出るまでに時間がかかる
学習記録や達成状況が媒体ごとに分かれるため、全体としてどれだけ取り組んだかを把握しづらい。
疲れている状態では、その日に何をどれだけ学習するかを考えること自体も負担になりやすい
								⬇︎
「頑張っているのに成果が出ていない」と感じてしまい、継続の意欲が下がってしまう
```
以上のことから、複雑な管理をしなくても、その日に学習に向き合えた行動や小さな前進を記録・可視化し、前向きに感じられる仕組みが重要だと考えます。

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー
働きながら、TOEIC・プログラミング・資格学習などのスキルアップに取り組んでいる社会人を想定しています。

特に対象にしたいのは、教材や学習アプリはすでに使っているものの、仕事の疲れや日々の予定によって学習の着手が後回しになりやすく、「今日もできなかった」と自己嫌悪になりやすい人です。

年齢設定：
- 年齢：20代前半から30代前半の社会人
- 理由：キャリア初期から中堅に向かう時期で、転職・昇進・業務スキル向上のために自主学習を始める動機が生まれやすい。一方で、日々の業務で疲れやすく、学習を継続する仕組みが必要になりやすいと考えたため。

生活・状況設定：
- 平日は仕事や家事でまとまった学習時間を取りづらい
- 出勤前、通勤時間、昼休み、帰宅後などの短い時間で学習することが多い
- 仕事終わりは疲れており、学習内容を考えること自体が負担になりやすい
- 週末にまとまった時間を取り、平日に進められなかった分を補うことがある

環境設定：
- 基本的に一人で学習している
- TOEICアプリ、動画教材、問題集、プログラミング学習サービスなど、学習内容そのものは既存の教材で行っている
- Notion、Todoアプリ、カレンダーなどで予定管理を試したことはあるが、管理すること自体が負担になり、続かなくなることがある

このサービスでは、教材アプリや学習サービスを置き換えるのではなく「今日は学習に着手できた」「短時間でも前に進めた」という行動を記録・可視化することを目的とします。

### 4-2. 自分とユーザーの距離

想定ユーザーとの距離：過去の自分、現在の自分、身近な友人

私自身、自主的に英語学習やプログラミング学習に取り組む中で、仕事の疲れや他の予定を理由に学習を後回しにしてしまった経験があります。また、学習計画やTodoアプリを使っていても、予定通りにできなかった時に罪悪感が残り、継続のモチベーションが下がることがありました。

身近にも、働きながら資格取得やスキルアップのために学習している人がいるため、
「学習を始めるまでの心理的ハードル」や「短時間でも前に進めた実感の得づらさ」に焦点を当ててフィーバックを得ながらサービス設計ができると考えています。

### 4-3. 実際に使われる可能性について（需要・存在確認）

このアプリを思い出す場面は、以下のようなタイミングを想定しています。
- 仕事終わりに「今日は勉強する予定だったのに、スマホを見ていたら時間が過ぎてしまった」と感じたとき
- 学習を始めたものの、数日で途切れて自己嫌悪になったとき
- Notion、Todoアプリ、カレンダーで予定を立てても、実行直前の心理的ハードルを越えられなかったとき
- 教材アプリの正答率やスコアだけでは、自分が積み上げた努力を実感しづらいと感じたとき

今の生活の中で代わりに使われているものは、主に以下の2種類だと想定します。
- 専用学習アプリ：正答率、間違いの指摘、問題演習など、学習内容に対するフィードバックを行う
- タスク管理アプリ：予定、期日、タスクの書き出し、進捗管理を行う

ただし、これらのアプリは「疲れている中でも5分だけ始められた」「予定通りではないが、少しでも学習に向き合えた」といった行動そのものを前向きに評価する仕組みは弱いと感じています。


学習内容は既存の教材で行い、このサービスでは着手・継続・小さな前進を記録し、学習を続けるための心理的ハードルを下げます。

### 4-4. ユーザーがサービスを導入して利用するまでのイメージ（導入・継続の流れ）

このサービスは、学習内容を提供するアプリではなく、学習に取り組む行動を支えるアプリとして利用されることを想定しています。

利用の流れは以下の通りです。
```text
1. 週末や空いている時間に、ざっくり学習予定日を決める
2. アプリを開き、カレンダーUIから学習日と学習内容を入力する
3. 学習日にアプリを開き、その日の体力に合わせてタイマーを選ぶ
   選択肢: 5分、10分、25分、60分
4. 開始ボタンを押す
   実際の学習は、TOEICアプリ、動画教材、問題集、書籍などで行う
5. 学習後に完了ボタンを押す
6. 学習時間、予定通りに開始できたか、タイマーを完了できたかをもとにポイントを獲得する
7. ポイントが貯まると、事前に自分で設定したご褒美と交換できる
   例: 300pt到達で「週末にカフェでケーキを食べる」
```
このサービスを継続して使う理由は、完璧に毎日続けることを前提にしない点にあります。

毎日長時間学習できたかではなく、ユーザーが決めた日に少しでも着手できたかを評価します。疲れている日に5分だけでも始められた行動をポイント化することで、「できなかった」ではなく「少しでも進めた」と受け取れるようにします。

将来的には、学習記録データをもとに、ユーザー自身が「どの時間帯に学習しやすいか」「どの曜日にサボりやすいか」「どのくらいの学習時間なら継続しやすいか」を確認できるようにしたいと考えています。

- どんなことができるか：
    - todoをクリアするとポイントがもらえる
    - 自分で設定したご褒美と交換できる

### 5-2. それでも自分が作りたい理由

【他の人と交流すること自体が負担】
みんチャレは、同じ目標を持つ人とチームを組み、毎日成果を報告し合うことで習慣化を支えるサービスです。「誰かに報告する」という強制力が継続の支えになる一方で、働きながら学習している社会人にとっては、学習に取り組むだけでなく、写真や内容を投稿したり、他の人と交流したりすること自体が負担だと感じました。

【評価の中心は「タスクを完了したかどうか」のみ】
MissionTodoは、todoをミッションとして管理し、達成するとポイントを獲得できるサービスです。タスク管理にゲーム要素を取り入れている点は近いですが、評価の中心は「タスクを完了したかどうか」です。そのため、「疲れている中で5分だけ学習を始められたこと」「予定通りではなくても少し前に進めたこと」「学習時間を積み重ねたこと」などは実感しづらいと感じました。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
