## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

このアイデアの原点は、自分自身が英語学習やプログラミング学習を１人でコツコツと続けてきた経験にあります。

私は、留学やワーキングホリデーをきっかけに、学習計画やTodoアプリを利用してタスク管理を行い英語学習に取り組んできました。

一人暮らしの社会人時代は、仕事や家事と並行しながら終業後や週末の隙間時間を利用して英語学習を行い、ワーホリ時代は慣れない海外で、不安や孤独になりながらも継続的にプログラミング学習を行なってきました。

しかし、英語やプログラミングのような学習は、必死に学習計画を立てて隙間時間で勉強したからといって、すぐに大きく伸びるものではなく成果もすぐに見えず、前に進んでいる実感が持てず、継続のモチベーションの管理が難しい経験をしました。

その中で、以下のように目標を達成できたら自分へご褒美を設定することが学習継続のモチベーションの１つとして有効でした。
```
例)
今週3回DMM英会話ができたら、友達とパブでビールを飲む
普段ならサボる時間帯に学習できたら、好きなクッキーを買う
```

---

### 2-2. なぜそれが「気になった」のか
- 自分の性格
	ゲームのように自分の成長が数字で可視化されることが楽しい
	完璧主義で、タスクを設定してできなかった時に罪悪感や自責の念を感じる

- 過去の経験
	小学校から高校までは、自主的に学習に取り組むという習慣が全くなかった。
	学習を始めた頃はサボったり、後回しにしたりしていたが、徐々に継続することで「できなかったこと」が、「できるようになる」瞬間に達成感を感じるようになった。
	「海外で自分自身で生活基盤を築く」「ヨーロッパを１人で旅行する」という、数年前の自分からは想像もできない体験をした。
- 大切にしている価値観
	１人で海外でも問題なく生活できるようになれたのは、コツコツと継続して工夫をしながら学習を続けた経験があるからだと信じており「工夫」をして継続することに価値があると感じている。

上記のような経験から、自分にとって学習とは、単に知識を増やすためのものではなく、自分の行動や工夫によって人生の選択肢を広げられるものだと感じています。

だからこそ、学習を継続するための「工夫」や、努力を前向きに受け取れる仕組みに価値があると考えています。

このように、「やろうと決めたことに取り組めた」「いつもならサボる時間に着手できた」という行動を評価し、ゲームのように前向きに続けられる仕組みをアプリとして作りたいと考えたのが、このアイデアの出発点です。

そのため次に、自分の体験やヒアリング内容をもとに、表面的に見えている困りごとと、本当に解決したい課題を整理します。

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

ヒアリングから、働きながら学習している社会人には以下のような困りごとがあると分かった。
```
- 学生のように、毎日同じ時間にまとまった学習時間を確保することが難しい
- 仕事の疲れから、「今日は勉強しよう」と思っていても後回しになりやすい
- 一人で学習しているため、周囲からの強制力が弱く、簡単にサボれる
- TOEIC専用アプリ、プログラミング教材、問題集、動画教材など、学習媒体が分散している
- Notion、Todoアプリ、Excel、カレンダーアプリは多機能だが、管理すること自体が負担になりやすい
- 目標を設定して学習に取り組んでも、期待したほど成果が見えず、落ち込むことがある
```

ヒアリング内容からより大きな問題は、忙しい中で学習に取り組んでいても、その努力や前進が実感しづらいことにあると考えた。

---
### 3-2. 本当に解決したい課題は何か

```
TOEIC、プログラミング、資格学習などは、成果が出るまでに時間がかかる
学習記録や達成状況が媒体ごとに分かれるため、全体としてどれだけ取り組んだかを把握しづらい。
疲れている状態では、その日に何をどれだけ学習するかを考えること自体も負担になりやすい
								⬇︎
「頑張っているのに成果が出ていない」と感じてしまい、継続の意欲が下がってしまう
```
以上のことから、複雑な管理をしなくても、その日に学習に向き合えた行動や小さな前進を記録・可視化し、前向きに感じられる仕組みが重要だと考えます。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
